# ARC Investor Deck Style Guide

A comprehensive visual and verbal identity guide for investor presentations, aligned with the Autonomous Resource Corporation website.

---

## Brand Essence

**Tagline:** Building the foundational platforms for the next industrial revolution: Open Intelligence and Digital Matter.

**Brand Personality:** Technical precision meets human aspiration. ARC communicates with the confidence of deep infrastructure expertise while maintaining accessibility for strategic investors.

- **Metrics:** Large numbers in white, labels in Muted Text, units uppercase

## Typography

### Font Stack

| Purpose | Font | Fallback | Weight |
|---------|------|----------|--------|
| **Display Headlines** | Newsreader | Georgia, serif | 400 (Regular), 400 Italic |
| **Body & UI** | Inter | system-ui, sans-serif | 400, 500, 600 |
| **Data & Code** | IBM Plex Mono | monospace | 400, 500 |

### Type Scale

| Element | Font | Size | Weight | Style |
|---------|------|------|--------|-------|
| Hero Headline | Newsreader | 72–96px | 400 | Mixed: Roman + Italic |
| Section Title | Newsreader | 48–56px | 400 | Roman |
| Slide Headline | Inter | 32–40px | 600 | — |
| Subheadline | Inter | 24px | 500 | — |
| Body Text | Inter | 18px | 400 | — |
| Caption/Label | Inter | 12–14px | 500 | Uppercase, tracking +0.1em |
| Data/Metrics | IBM Plex Mono | 48–64px | 500 | — |
| Code/Terminal | IBM Plex Mono | 14px | 400 | — |

### Typography Rules

- **Headlines:** Use Newsreader for emotional impact. Alternate between roman and italic within a single headline for visual interest (e.g., "Augmenting *Ingenuity*")
- **Body:** Inter for all explanatory text. Keep line lengths under 75 characters.
- **Metrics:** IBM Plex Mono for all numbers, percentages, and technical data
- **Labels:** Always uppercase with letter-spacing for category tags (e.g., `DEFENSE`, `ENERGY`, `MEDICINE`)
- **Never:** Use Newsreader for body text or IBM Plex Mono for headlines

## Voice & Tone

### Overall Character

**Tone:** Authoritative yet aspirational. Technical precision without jargon fatigue.

**Influences:** The clarity of Doug Engelbart's "augmenting human intellect" vision. Infrastructure confidence, not startup hype.

### Language Principles

- **Sentence length:** Medium (15–25 words). Shorter for impact statements.
- **Jargon level:** Accessible to infrastructure/deep-tech investors. Define novel terms on first use.
- **Active voice:** Yes. "ARC builds..." not "Infrastructure is built by ARC..."
- **Person:** "ARC" (third person) for company actions. "You/your" sparingly for investor benefits.

### Preferred Phrasing

| Instead of... | Use... |
|---------------|--------|
| data center | AI factory |
| GPU cluster | AI compute infrastructure |
| 3D printing | additive manufacturing |
| superalloy | advanced materials |
| machine learning | AI (when general) |
| build | engineer / manufacture |
| partnership | collaboration / mission |

### Company-Specific Terms

| Term | Definition | Usage Note |
|------|------------|------------|
| **ARCNet** | ARC's distributed network of AI factory sites | Always capitalized. One word. |
| **ADAM** | Autonomous Discovery of Advanced Materials platform | Acronym only after first full mention |
| **AI-to-atom** | End-to-end integration from AI compute to physical manufacturing | Hyphenated. Defines ARC's unique position. |
| **AI factory** | Purpose-built 100MW+ AI compute facility | Lowercase "factory" |
| **Open Intelligence** | ARC's approach to democratized AI infrastructure | Capitalized as brand pillar |
| **Digital Matter** | Physical manufacturing enabled by AI design | Capitalized as brand pillar |

### Forbidden Language

Never use these words—they signal hype over substance:

- "Revolutionary" / "Disruptive" / "Game-changing"
- "Best-in-class" / "World-class" / "Industry-leading"
- "Synergy" / "Leverage" (as verb)
- "Cutting-edge" / "Bleeding-edge"
- "Paradigm shift"
- "Unlock" / "Unleash"

### Aspirational Alternatives

| Instead of... | Try... |
|---------------|--------|
| Revolutionary | Foundational |
| Disruptive | Essential |
| Game-changing | Infrastructure-grade |
| Cutting-edge | Purpose-built |

## Visual Elements

### Grid Overlay

The website uses a subtle technical grid pattern as a background texture.

- **Application:** Use on cream/light backgrounds to add technical depth
- **Opacity:** 5–10% maximum
- **Style:** Fine lines, engineering drawing aesthetic

### Status Indicators

Borrowed from terminal/systems UI:

```
● OPERATIONAL    (Green #22C55E)
● RUNNING        (Green #22C55E + label)
○ PENDING        (Gray outline)
```

### Buttons & CTAs

| Type | Style |
|------|-------|
| **Primary** | ARC Violet (#5E5CE6) fill, white text, rounded corners (8px) |
| **Secondary** | Transparent, white border, white text |
| **Navigation** | Text only, uppercase, tracking +0.05em |

### Cards

- **Dark cards:** #111111 background, subtle border, slight rounded corners
- **Light cards:** White background on cream, thin gray border
- **Hover:** Subtle violet glow or border highlight

### Iconography

- **Style:** Minimal, geometric, single-weight line icons
- **Size:** 24px standard, 16px inline
- **Color:** Match text color or ARC Violet for interactive

## Data Visualization

### Chart Style

- **Background:** Transparent or slide background color
- **Grid lines:** Minimal, 10% opacity gray
- **Data colors:** ARC Violet primary, Cyan secondary, Status Green tertiary
- **Labels:** IBM Plex Mono, uppercase for axis labels

### Metric Callouts

Format large statistics consistently:

```
92%
CHINA'S CONTROL
```

- Number: IBM Plex Mono, 48–64px, white
- Label: Inter, 12px, uppercase, Muted Text (#9CA3AF), tracking +0.1em

### Preferred Chart Types

- **Comparisons:** Horizontal bar charts
- **Timelines:** Linear milestone visualization
- **Market sizing:** Nested circles (TAM/SAM/SOM) or simple bars
- **Progress:** Linear progress bars with Status Green

### Avoid

- Pie charts (except use-of-funds breakdown)
- 3D effects
- Excessive gradients
- Complex multi-axis charts

## Slide Guidelines

### Content Limits

| Element | Maximum |
|---------|---------|
| Slide title | 6 words |
| Headline | 10 words |
| Bullets per slide | 3 |
| Words per bullet | 12 |
| Stat callout | Number + 4-word label |

### Slide Types

**Title Slide**
- "Autonomous Resource Corporation" in Inter 600
- Tagline in Newsreader italic
- Deep Black background
- No funding details on title

**Section Dividers**
- Large Newsreader headline, mixed roman/italic
- Minimal supporting text
- Full-bleed imagery or Deep Black

**Content Slides**
- Slide title: Inter 600, top-left
- 3 bullet maximum
- Leave 40% negative space minimum

**Data Slides**
- Can use Warm Cream background for contrast
- Grid overlay permitted
- Metrics prominent, explanatory text minimal

**Team Slide**
- Headshots: High-quality, consistent treatment
- Name: Inter 600
- Title + credential: Inter 400, one line each

### Citation Style

- **In-deck:** [Source: Document Name] — Inter 12px, Muted Text, bottom-right
- **Data freshness:** 2024–2026 data only. Flag if older.

## Numbers & Metrics

### Formatting Rules

| Type | Format | Example |
|------|--------|---------|
| Currency | Symbol + number + unit | $1.2B |
| Large numbers | Abbreviated with unit | 150 TWh |
| Percentages | Whole numbers | 92% (not 92.3%) |
| Time | Number + unit | 3 Days |
| Ranges | En-dash, no spaces | 100–250MW |

### Currency

- Primary: USD
- Symbol before number: $1.2B
- Spell out "billion" only in prose, never on slides

## Imagery Guidelines

### Photography Style

- **Aesthetic:** Industrial precision, slight desaturation, high contrast
- **Subjects:** Manufacturing facilities, AI infrastructure, materials science
- **Treatment:** Dark vignettes acceptable, avoid over-processing
- **Avoid:** Stock photo clichés, handshakes, generic tech imagery

### Image Generation (AI)

When generating imagery:

- Color palette: Deep blacks, violet accents, cyan highlights
- Style: Technical, architectural, slightly futuristic
- Subjects: Data centers, advanced manufacturing, materials visualization
- Text in images: Minimal. Large sans-serif if required.
- Avoid: Photorealistic faces, generic "AI brain" imagery

## Competitor References

- Name competitors on competition/positioning slide only
- Position ARC in context of "Open + Industrial Execution" quadrant
- Relevant comparisons: OpenAI, AWS, Google Cloud, Anthropic (compute), established aerospace materials suppliers (manufacturing)
- Never disparage—position ARC's differentiation
